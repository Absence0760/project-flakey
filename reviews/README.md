# reviews/

Working drop-zone for audit and persona reports generated by Claude auditor
agents (`/audit/*`, `/persona`, `/fanout-review`, etc.).

## How this folder works

- **Only this `README.md` is tracked by git.** Everything else in `reviews/`
  is gitignored (see the `reviews/` block in the repo root `.gitignore`).
- Auditor agents write their findings here as `<scope>.md` /
  `<persona>.md` so reports persist on disk and you can pick them up across
  multiple Claude Code sessions.
- Reports are **working documents, not the source of truth.** They go stale
  fast. Apply findings as path-scoped commits / PRs, then let the report file
  be overwritten or deleted on the next run.

## Convention

- One file per audit slice or persona, e.g. `auth.md`, `persona-admin.md`.
- Re-running an auditor over the same scope overwrites its file.
- Nothing here is shared via git — sync across sessions happens through the
  local working tree.
